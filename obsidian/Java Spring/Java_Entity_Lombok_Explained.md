# Java Entities & Lombok: The "Model" Layer
*Notes on Modern Java Data Structures*

## 1. Is this like a Python Model?
**YES.**
If you have used **Django Models** or **SQLAlchemy** in Python, this is exactly the same concept.

*   **Python (Django):** You define a `class User(models.Model)` and fields like `username = models.CharField(...)`.
*   **Java (JPA):** You define a `class User` annotated with `@Entity`.

**The Purpose:** It maps a **Java Class** directly to a **Database Table**.
*   `@Table(name = """user""")` = "This class represents the `user` table in Postgres."
*   `@Column(name = "f_name")` = "This field maps to the `f_name` column."

---

## 2. What is the "Annotation Magic"? (Lombok)
In the old days (Legacy Java), for a class with 10 fields, you had to write 10 "Getter" methods, 10 "Setter" methods, a `toString()`, an `equals()`, and 3 constructors. That's 100+ lines of clutter.

**Lombok** (the library providing these annotations) automates this at **compile time**.

### `@Data`
*   **What it does:** Automatically generates **Getters** (`getUsername()`), **Setters** (`setUsername()`), `toString()`, `equals()`, and `hashCode()`.
*   **Why use it:** Keeps your code clean and readable.

### `@NoArgsConstructor`
*   **What it does:** Generates a "blank" constructor: `public User() {}`.
*   **Why it's needed:** **JPA/Hibernate REQUIRES this.** When Hibernate fetches data from the DB, it first creates an empty object using this constructor, then fills in the fields.

### `@AllArgsConstructor`
*   **What it does:** Generates a constructor with *all* arguments.

### `@Builder`
*   **What it does:** Implements the **Builder Pattern**.
*   **How you use it:**
    ```java
    // Instead of: new User(1, "job", "emp", "user", "first"...) // Hard to read!
    
    // You write:
    User.builder()
        .firstName("Tan")
        .email("tan@gmail.com")
        .role(Role.HR)
        .build();
    ```
*   **Connection to your file list:** You saw `UserDTO$UserDTOBuilder.class`. This is the **compiled bytecode** for the helper class that Lombok secretly created to make `.builder()` work. The `$` sign means it is an *inner class*.

## 3. What are those `.class` files?
You mentioned seeing paths like `target\classes\...\DataSeederConfig.class`.

*   **.java files:** The human-readable source code you write (in `src/main/java`).
*   **.class files:** The machine-readable **Bytecode** that the Java Compiler (`javac`) produces.
*   **target/ folder:** This is where Maven dumps all the compiled files. You **never** edit files in `target`.

### Specific Files you asked about:
*   `DataSeederConfig.class`: This is likely a new Configuration class Antigravity added to handle database seeding (populating initial data) using Spring Beans instead of the old manual `main()` method.
*   `UserDTO$UserDTOBuilder.class`: As mentioned above, this is the "ghost" helper class generated by Lombok's `@Builder` annotation.
